# chatgpt小tips
1. 购买腾讯云美国轻量应用服务器
   
2. 本地Mac电脑生成rsa类型密钥对
可以先查看是否已存在，已存在就不用了/Users/xxxx/.ssh/id_rsa
```
ssh-keygen -t rsa
```

ssh-keygen 是一个用于生成、管理和转换身份验证密钥的工具，它通常用于创建 SSH 协议中使用的公钥和私钥对。在这个代码中，我们使用 ssh-keygen 命令生成一个 RSA 类型的密钥对。

具体来说，ssh-keygen -t rsa 命令会执行以下操作：

提示用户输入要保存密钥的文件名和路径（默认为 id_rsa）。
要求用户输入一个可选的密码短语，用于保护私钥（如果不提供，私钥将不加密）。
生成一个新的 RSA 密钥对（包括公钥和私钥）。
将公钥保存到.pub文件中，将私钥保存到指定的文件中（默认为id_rsa`）。
执行此命令后，您将拥有一个 RSA 类型的 SSH 密钥对，可以用于安全地访问支持 SSH 协议的服务器和服务。

3. 将公钥复制到远程服务器
```
# 将公钥复制到远程服务器
ssh-copy-id user@your_server_ip
# 输入远程服务器的密码
```
ssh-copy-id 是一个用于将本地 SSH 公钥复制到远程服务器上对应用户的 authorized_keys 文件中的命令。这样可以实现免密码登录。

user 是远程服务器上的用户名，your_server_ip 是远程服务器的 IP 地址。

执行这个命令后，系统会提示你输入远程服务器上用户的密码。输入正确的密码后，本地的 SSH 公钥将被复制到远程服务器的 authorized_keys 文件中。之后，当你使用 SSH 连接到远程服务器时，只要提供正确的用户名，就可以直接登录，而无需输入密码。

4. 本地Mac启动转发端口，端口最好改一下

```
vim /Users/xxxx/chatgpt_ssh_tunnel_check.sh
```

```
#!/bin/bash

CMD="ssh -N -D 1080 用户名@服务器IP地址"

pgrep -f "$CMD" > /dev/null 2>&1

if [ $? -ne 0 ]; then
    echo "Starting SSH Tunnel..."
    $CMD &
else
    echo "SSH Tunnel is already running."
fi
```

5. 本地Mac添加定时任务
```
crontab -e
```

```
*/1 * * * * /bin/bash /Users/xxxx/chatgpt_ssh_tunnel_check.sh
```

ssh -N -D 1080 用户名@服务器IP地址
在这段代码中,主要使用了SSH协议登录到远程服务器。其中：
-N 参数表示不在远程服务器上执行命令，即只用于端口转发。
-D 1080 参数表示将本地主机的1080端口转发到远程服务器上的某个端口。
user@your_server_ip 参数表示登录远程服务器的用户名和服务器IP地址。
通过这段代码，可以在本地主机上通过SSH协议访问远程服务器上的资源，并且可以通过将本地主机的端口转发到远程服务器来实现某些网络功能。
